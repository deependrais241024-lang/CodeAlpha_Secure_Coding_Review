# 🔐 Secure Coding Review – CodeAlpha Cyber Security Internship (Task 3)

## 📌 Project Overview

This project was completed as part of the CodeAlpha Cyber Security Internship. The objective is to identify common security vulnerabilities, understand insecure coding practices, and demonstrate secure coding techniques to improve software security.

## 🎯 Objectives

* Understand secure coding principles.
* Identify common software vulnerabilities.
* Compare insecure and secure coding examples.
* Learn best practices for writing secure applications.

## ⚠️ Vulnerabilities Covered

* SQL Injection
* Weak Authentication
* Hardcoded Credentials
* Improper Input Validation

## 🛠️ Tools Used

* Python
* GitHub
* Google Docs
* Cyber Security Concepts

## 📂 Repository Structure

* `Secure_Coding_Review_Report.pdf` – Detailed project report.
* `sample-code/insecure_login.py` – Example of insecure coding.
* `sample-code/secure_login.py` – Example of secure coding.

## 📖 Secure Coding Best Practices

* Validate all user input.
* Avoid hardcoded credentials.
* Use secure authentication methods.
* Follow the principle of least privilege.
* Keep software and dependencies updated.

## 📊 Outcome

This project demonstrates how secure coding practices help reduce software vulnerabilities and improve application security.
